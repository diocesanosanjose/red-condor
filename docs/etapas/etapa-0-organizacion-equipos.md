# Etapa 0 -- Organización de Equipos de Investigación

> **Objetivo:** Comprender el problema, investigar tecnologías y analizar posibles soluciones antes de construir prototipos.

---

## Equipos de investigación

### Equipo 1: Comunicaciones

Comprender cómo transmitir información en zonas rurales o de montaña donde no existe conectividad tradicional.

| Tema | Descripción |
|------|-------------|
| **LoRa** | Protocolo de largo alcance y bajo consumo |
| **Radioenlaces** | Transmisión punto a punto sin infraestructura |
| **Antenas** | Tipos, direccionalidad y alcance |
| **Cobertura** | Factores geográficos que afectan la señal |
| **Repetidores** | Extensión del alcance en terreno difícil |

### Equipo 2: Energía

Analizar cómo alimentar dispositivos tecnológicos en lugares alejados de la red eléctrica convencional.

| Tema | Descripción |
|------|-------------|
| **Paneles solares** | Tipos, eficiencia y orientación para máxima captación |
| **Baterías** | Almacenamiento energético y tecnologías disponibles |
| **Consumo energético** | Optimización del gasto en dispositivos remotos |
| **Autonomía** | Cálculo de operación sin recarga externa |

### Equipo 3: Hardware

Investigar los componentes físicos necesarios para construir los dispositivos del proyecto.

| Tema | Descripción |
|------|-------------|
| **ESP32** | Microcontrolador principal: procesamiento, WiFi y Bluetooth integrados |
| **Sensores** | Temperatura, humedad, presión y otros parámetros ambientales |
| **GPS** | Módulos de posicionamiento para ubicación en tiempo real |
| **Carcasas** | Diseño resistente a condiciones climáticas adversas |

### Equipo 4: Usuarios

Comprender quiénes utilizarían el sistema y qué problemas concretos necesitan resolver en su contexto real.

| Tema | Descripción |
|------|-------------|
| **Puesteros** | Productores rurales aislados que necesitan comunicación básica y monitoreo ambiental |
| **Rescatistas** | Equipos de rescate que operan en zonas sin cobertura celular |
| **Defensa Civil** | Organismos que requieren datos en tiempo real para la toma de decisiones |
| **Necesidades reales** | Identificación de problemas concretos que el sistema debe resolver |

### Equipo 5: Infraestructura

Analizar cómo almacenar, procesar y visualizar la información generada por el sistema.

| Tema | Descripción |
|------|-------------|
| **Servidores** | Infraestructura de procesamiento central |
| **Bases de datos** | Registro histórico y consultas eficientes |
| **Aplicaciones web** | Acceso desde cualquier dispositivo |
| **Monitoreo** | Alertas y visualización en tiempo real |

---

## Resultado esperado

Cada equipo deberá completar las siguientes actividades y compartir sus hallazgos con el resto del curso:

1. **Investigar los conceptos asignados** -- Explorar cada tema del área correspondiente con fuentes confiables.
2. **Elaborar un resumen de hallazgos** -- Documentar los conceptos clave de forma clara y ordenada.
3. **Identificar dudas y preguntas abiertas** -- Registrar lo que aún no está claro para discutir en clase.
4. **Compartir con el resto del curso** -- Presentar los resultados de manera que todos los equipos aprendan.
5. **Proponer aplicaciones para Red Condor** -- Conectar la investigación con posibles soluciones del proyecto.

---

## Entregables por equipo

Cada equipo debe crear una carpeta en `docs/etapas/equipo-X-nombre/` con:

```
equipo-X-nombre/
├── README.md              # Resumen general del equipo
├── investigacion.md        # Investigación detallada de cada tema
├── preguntas-abiertas.md   # Dudas y preguntas pendientes
└── propuestas.md           # Aplicaciones propuestas para Red Condor
```
