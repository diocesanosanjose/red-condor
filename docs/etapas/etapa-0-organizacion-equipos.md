# Etapa 0 -- Organizacion de Equipos de Investigacion

> **Objetivo:** Comprender el problema, investigar tecnologias y analizar posibles soluciones antes de construir prototipos.

---

## Equipos de investigacion

### Equipo 1: Comunicaciones

Comprender como transmitir informacion en zonas rurales o de montana donde no existe conectividad tradicional.

| Tema | Descripcion |
|------|-------------|
| **LoRa** | Protocolo de largo alcance y bajo consumo |
| **Radioenlaces** | Transmision punto a punto sin infraestructura |
| **Antenas** | Tipos, direccionalidad y alcance |
| **Cobertura** | Factores geograficos que afectan la senal |
| **Repetidores** | Extension del alcance en terreno dificil |

### Equipo 2: Energia

Analizar como alimentar dispositivos tecnologicos en lugares alejados de la red electrica convencional.

| Tema | Descripcion |
|------|-------------|
| **Paneles solares** | Tipos, eficiencia y orientacion para maxima captacion |
| **Baterias** | Almacenamiento energetico y tecnologias disponibles |
| **Consumo energetico** | Optimizacion del gasto en dispositivos remotos |
| **Autonomia** | Calculo de operacion sin recarga externa |

### Equipo 3: Hardware

Investigar los componentes fisicos necesarios para construir los dispositivos del proyecto.

| Tema | Descripcion |
|------|-------------|
| **ESP32** | Microcontrolador principal: procesamiento, WiFi y Bluetooth integrados |
| **Sensores** | Temperatura, humedad, presion y otros parametros ambientales |
| **GPS** | Modulos de posicionamiento para ubicacion en tiempo real |
| **Carcasas** | Diseno resistente a condiciones climaticas adversas |

### Equipo 4: Usuarios

Comprender quienes utilizarian el sistema y que problemas concretos necesitan resolver en su contexto real.

| Tema | Descripcion |
|------|-------------|
| **Puesteros** | Productores rurales aislados que necesitan comunicacion basica y monitoreo ambiental |
| **Rescatistas** | Equipos de rescate que operan en zonas sin cobertura celular |
| **Defensa Civil** | Organismos que requieren datos en tiempo real para la toma de decisiones |
| **Necesidades reales** | Identificacion de problemas concretos que el sistema debe resolver |

### Equipo 5: Infraestructura

Analizar como almacenar, procesar y visualizar la informacion generada por el sistema.

| Tema | Descripcion |
|------|-------------|
| **Servidores** | Infraestructura de procesamiento central |
| **Bases de datos** | Registro historico y consultas eficientes |
| **Aplicaciones web** | Acceso desde cualquier dispositivo |
| **Monitoreo** | Alertas y visualizacion en tiempo real |

---

## Resultado esperado

Cada equipo debera completar las siguientes actividades y compartir sus hallazgos con el resto del curso:

1. **Investigar los conceptos asignados** -- Explorar cada tema del area correspondiente con fuentes confiables.
2. **Elaborar un resumen de hallazgos** -- Documentar los conceptos clave de forma clara y ordenada.
3. **Identificar dudas y preguntas abiertas** -- Registrar lo que aun no esta claro para discutir en clase.
4. **Compartir con el resto del curso** -- Presentar los resultados de manera que todos los equipos aprendan.
5. **Proponer aplicaciones para Red Condor** -- Conectar la investigacion con posibles soluciones del proyecto.

---

## Entregables por equipo

Cada equipo debe crear una carpeta en `docs/etapas/equipo-X-nombre/` con:

```
equipo-X-nombre/
├── README.md              # Resumen general del equipo
├── investigacion.md        # Investigacion detallada de cada tema
├── preguntas-abiertas.md   # Dudas y preguntas pendientes
└── propuestas.md           # Aplicaciones propuestas para Red Condor
```
